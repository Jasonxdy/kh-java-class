Hello, JSP world!

JSP (Java Server Page)란, Java 코드가 들어가 있는 HTML 코드를 말하며 서블릿의 복잡함을 간단하게 작성, 해결할 수 있다.
JSP의 가장 큰 장점은 Servlet에서 비즈니스 로직(Java)과 프레젠테이션 로직(HTML)을 분리할 수 있다는 것이다.
프로그래머가 직접 Servlet에 코딩한 것보다 더욱 최적화된 형태의 Servlet을 생성해줌.
웹 애플리케이션 상에서 변수의 사용 가능한 범위 설정이 쉽다.

JSP - 1

스크립팅 원소(Scripting Element)는 JSP 페이지에서 자바 코드를 직접 기술할 수 있게 해주는 기능.
1. <%! %> (선언문) : 멤버변수, 메소드를 선언하기 위해 사용된다.
2. <% %> (스크립틀릿, scriptlet) : JSP에서 자바코드 사용을 가능하게 해줌. 3. <%= %> (표현식, expression) : Java에서 작성한 변수, 결과값을 출력하기 위해 사용된다.

JSP - 2

페이지 지시자
JSP 맨 윗 부분에는 JSP 파일의 속성을 기술하는 page 지시어(지시자)가 존재함.
language : 사용할 스클비트 언어 유형 지정
contentType : 웹 브라우저가 받아 볼 페이지의 형식, 인코딩 지정
pageEncoding : JSP 파일에 기록된 자바코드의 인코딩 방식 지정
import : Java의 import와 같이 타 클래스 사용 시 추가
errorPage : 해당 JSP에서 오류 발생 시 이동할 에러 페이지 지정
isErrorPage : 해당 JSP가 오류 처리 전용 페이지임을 지정

JSP - 3

include
정적 include 방식으로 페이지 내에 include한 소스 코드가 단순 텍스트로 그대로 복사되어 포함된다.
include한 페이지 코드 내부에 변수가 선언되어 있다면 베이스가 되는 페이지의 변수와 충돌이 날 수 있으므로 유의해야함.
include 페이지 내의 import는 베이스와 다른 include 페이지와 공유된다.
include는 페이지를 조각내어 필요한 부분에 삽입할 때 주로 사용된다.
